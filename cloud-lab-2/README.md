## 1. Цель работы

Получение навыков аналитики и понимания спектра публичных облачных сервисов без привязки к вендору. Формирование у
студентов комплексного видения Облака.

## 2. Исходные данные

Для анализа был предоставлен слепок биллинговых данных Microsoft Azure в формате CSV, содержащий записи о потреблении
различных сервисов с полями:

- Meter Category и Meter Sub-Category — категория и подкатегория учета потребления
- Meter Name — конкретное название тарифицируемой метрики
- Consumed Service — используемый сервис Azure (например, Microsoft.DBforPostgreSQL)

## 3. Ход работы

В рамках задания была применена единая сервисная модель, разработанная в первой лабораторной работе для AWS.
Пятиуровневая иерархия IT Tower → Service Family → Service Type → Service Sub Type → Service Usage Type была наложена на
данные Microsoft Azure.

Ключевым принципом работы было определение основной функции сервиса Azure и отнесение его к уже существующей категории
из модели для AWS. Например:

* **Azure Database for PostgreSQL** был классифицирован как **Database**, поскольку является управляемой базой данных,
  аналогично другим сервисам этой категории в модели AWS, таким как **Amazon QLDB**.

В результате была создана единая таблица, в которой сервисы AWS и Azure описаны в согласованных категориях. Это
позволяет проводить анализ затрат и потребления ресурсов на уровне бизнес-функций, таких как вычисления, хранение,
аналитика.

В биллинге были выявлены и классифицированы ключевые группы сервисов:

- **Управляемые реляционные базы данных** (Azure Database for PostgreSQL, Azure Database for MySQL) <br />
  Это полностью управляемые службы для запуска, масштабирования и администрирования реляционных баз данных в облаке. Они
  избавляют пользователей от задач, связанных с управлением инфраструктурой, установкой исправлений и резервным
  копированием, позволяя сосредоточиться на разработке приложений и данных. <br />
  Аналоги: Yandex Managed Service for PostgreSQL/MySQ в Yandex Cloud, Cloud Databases в VK Cloud.
- **Платформы для анализа данных и инженерии** (Azure Databricks, Azure HDInsight) <br />
  Azure Databricks — это платформа аналитики на базе Apache Spark, оптимизированная для совместной работы в области
  данных, машинного обучения и аналитики. Azure HDInsight — это управляемая облачная служба для аналитики с открытым
  исходным кодом, которая поддерживает такие фреймворки, как Hadoop, Spark и Kafka. <br />
  Аналоги: Yandex Data Processing, VK Cloud Big Data.
- **Сервисы интеграции данных** (Azure Data Factory) <br />
  Это полностью управляемая служба интеграции данных, которая позволяет создавать, планировать и оркестрировать процессы
  извлечения, преобразования и загрузки и извлечения, загрузки и преобразования данных в гибридной среде. <br />
  Аналоги: не имеет прямого российского аналога.
- **Потоковая аналитика в реальном времени** (Azure Stream Analytics) <br />
  Служба для обработки и анализа больших объемов потоковых данных в реальном времени, поступающих с устройств Интернета
  вещей, датчиков, веб-сайтов и приложений, с использованием языка, похожего на SQL. <br />
  Аналоги: Yandex Data Streams.
- **Сервисы мониторинга и уведомлений** (Azure Monitor) <br />
  Платформа для мониторинга производительности и доступности приложений и инфраструктуры. Она собирает и анализирует
  данные телеметрии (метрики, журналы) и может отправлять уведомления (по электронной почте, через веб-перехватчики) о
  критических событиях. <br />
  Аналоги: Yandex Monitoring, Cloud Monitoring в VK Cloud.
- **Службы резервного копирования и восстановления** (Azure Backup) <br />
  Это централизованная служба для защиты данных, работающая в гибридной среде. Она позволяет создавать резервные копии
  виртуальных машин, баз данных, файловых серверов и обеспечивает централизованное управление и мониторинг. <br />
  Аналоги: VK Cloud Backup.
- **Службы кэширования в памяти** (Azure Cache for Redis) <br />
  Управляемая служба на базе Redis, которая предоставляет высокопроизводительный кэш данных в памяти для ускорения
  работы приложений и баз данных за счет сокращения задержек при обращении к данным. <br />
  Аналоги: Yandex Managed Service for Redis, Cloud Databases в VK Cloud.
- **Вычислительные ресурсы** (Виртуальные машины Azure, Azure Batch) <br />
  Виртуальные машины Azure предоставляют масштабируемые вычислительные ресурсы по запросу. Azure Batch — это служба для
  эффективного выполнения крупномасштабных параллельных и высокопроизводительных вычислительных заданий в облаке. <br />
  Аналоги: Yandex Compute Cloud, VK Cloud Compute.
- **Специализированные хранилища для аналитики** (Azure Data Lake Store) <br />
  Облачные хранилища, оптимизированные под хранение и обработку петабайтов структурированных и неструктурированных
  данных для выполнения задач аналитики, машинного обучения и работы с фреймворками Big Data (Hadoop, Spark). <br />
  Аналоги: Yandex Object Storage.
- **Сети доставки контента** (Azure CDN) <br />
  Географически распределённые сети серверов, которые кэшируют контент (изображения, видео, CSS/JS-файлы) на
  периферийных узлах, расположенных близко к пользователям, для значительного ускорения его загрузки и снижения нагрузки
  на основную инфраструктуру. <br />
  Аналоги: Yandex Cloud CDN.

## 4. Вывод

В результате выполнения лабораторной работы была создана единая кросс-провайдерная сервисная модель. <br />
Её ключевым результатом является демонстрация того, что сервисы AWS и Azure, решающие аналогичные бизнес-задачи, могут
быть последовательно описаны в рамках единой системы категорий, что формирует основу для их сравнения и анализа.
