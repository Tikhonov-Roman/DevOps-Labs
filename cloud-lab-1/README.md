## 1. Цель работы

Знакомство с облачными сервисами. Понимание уровней абстракции над инфраструктурой в облаке. Формирование понимания
типов потребления сервисов в сервисной-модели.

## 2. Исходные данные

Для анализа был предоставлен слепок биллинговых данных AWS в формате CSV, содержащий записи о потреблении различных
сервисов с полями:

- Product Code — код продукта AWS (например, AmazonS3)
- Usage Type — тип использования ресурса (например, %DataTransfer%)
- lineItem/Operation — операция (часто пустая)
- lineItem/LineItemDescription — описание строки счета

## 3. Ход работы

В рамках задания был проанализирован предоставленный файл облачного провайдера Amazon Web Services (наша команда выбрала
файл "Mapping Rules AWS team 10", руководствуясь тем, что в списке наша команда занимает 32 место, взяли остсаток от
деления на 11).
На основе анализа полей Product Code, Usage Type и других атрибутов выполнена структуризация потребляемых сервисов и
распределение их по пятиуровневой иерархии:
`IT Tower → Service Family → Service Type → Service Sub Type → Service Usage Type`.
В результате данные были структурированы, что позволяет однозначно отнести каждый элемент
расхода к определенному классу: Storage, Database, Networking, Cloud Services или к накладным расходам.

В биллинге были выявлены и классифицированы ключевые группы сервисов:
- **Объектное хранилище** (Amazon S3)
  Это масштабируемое и надежное объектное хранилище для хранения неструктурированных данных любого объема, таких как
  фото, видео, резервные копии и данные для аналитики. Оно предлагает высокую доступность, безопасность и гибкие классы
  хранения для оптимизации затрат.
  Аналоги: Yandex Object Storage, SberCloud Object Storage, VK Cloud S3.
- **Специализированные управляемые базы данных** (Amazon QLDB)
  Это управляемые базы данных. Они создают неизменяемый и криптографически проверяемый журнал всех изменений
  данных. Это нужно для приложений, где критически важны полный аудит и доверие к истории данных, например, в финансовых
  системах, цепях поставок или для ведения систем записей.
  Аналоги: Yandex Database в Yandex Cloud.
- **Аналитика и хранилище данных** (Amazon Redshift)
  Это облачное хранилище данных, предназначенное для выполнения высокопроизводительной аналитики на петабайтах
  структурированных и полуструктурированных данных с помощью SQL. Оно используется для бизнес-аналитики, создания
  отчетов, панелей управления и сложной аналитики в режиме, близком к реальному времени.
  Аналоги: Yandex DataProc, VK Cloud Big Data.
- **Сетевая инфраструктура и безопасность** (Amazon VPC)
  Amazon VPC позволяет создавать логически изолированные виртуальные сети в облаке AWS. В этой
  сети можно запускать ресурсы (например, виртуальные машины) с полным контролем над IP-адресацией, таблицами
  маршрутизации и сетевыми шлюзами. Это основа для обеспечения безопасности и подключения облачных ресурсов.
  Аналоги: Yandex Cloud Virtual Private Cloud, VK Cloud Network.
- **Сервисы коммуникаций и уведомлений** (Amazon SES, Amazon SNS)
  Amazon SES — это облачный сервис для отправки и получения электронной почты. Он нужен для
  массовых рассылок, транзакционных писем (например, пароли, счета), также он обеспечивает высокую доставляемость.
  Amazon SNS — это полностью управляемый сервис обмена сообщениями. Он нужен для
  отправки уведомлений (push, SMS, email) от приложений пользователям или для обмена сообщениями между
  распределенными компонентами приложений.
  Аналоги: Yandex Cloud Message Queue.

В результате выполнения лабораторной работы была создана сервисная модель потребления облачных сервисов AWS.
